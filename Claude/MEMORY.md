## Проекты

### Звезда Луки
- Папка: `Звезда Луки/`
- Тип: Изобретение — физическая игрушка-трансформер на LEGO Technic
- Идея: цельный объект на шаровых шарнирах, меняет форму через поворот Г-рычагов и скручивание секций — без снятия деталей
- Базовая форма: плоский крест (+), каркас из 4 чёрных Г-образных коннекторов, 4 бирюзовых блока по периметру
- 10 зафиксированных трансформаций (робот, птица, краб, буква O и др.)
- Уникальность: нет фиксированного сценария, ребёнок-изобретатель, бесконечные формы
- Аналоги: LEGO Bionicle, Klixx, Transformers (фирменные) — все уступают по гибкости
- Патент: не оформлен, нужен патентный поиск по классам A63H 33/00 и A63H 3/00
- Файлы: `concept.md`, `patent.md`, `info.md`, `context.md`

### Механика Ниндзя
- Папка: `Механика Ниндзя/`
- Тип: Теория — психология / жизнь / поведение
- Идея: состояние пустой головы открывает доступ к рефлексам (подсознанию) → они берут управление → удача/поток
- Метод входа: пение в Ниндзя (легче) или концентрация на ничто
- Три режима: момент / обучение (тело запоминает паттерны) / генерация (подсознание само придумывает действия)
- Эффект выхода: откат ~75% невезения после выхода из состояния — причина неизвестна
- Статус: теория оформлена, статья написана (article.md), эксперименты записаны но не проведены
- Файлы: `theory.md`, `sources.md`, `experiments.md`, `notes.md`, `article.md`, `context.md`

### Как собирать информацию для нового проекта
- Задавать 10+ вопросов через AskUserQuestion, не все сразу — по смыслу
- Всё что не понятно на 100% — спрашивать, не угадывать
- Использовать ВСЕ материалы которые даёт пользователь — ничего не игнорировать
- Если что-то из материалов не вписывается в шаблон — изменить шаблон или спросить

### Как писать документы
- Обычные файлы (concept.md, theory.md и т.п.) — живым языком, понятно, без бизнес-клише
- Официальные документы (patent.md, договоры) — официальным языком, всё ок
- Описывать проект максимально подробно, не сокращать
