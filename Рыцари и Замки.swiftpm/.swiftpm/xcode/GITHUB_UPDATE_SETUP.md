# Инструкция по настройке GitHub обновлений

## 🔧 Настройка

1. Откройте файл `GitHubUpdater.swift`
2. Найдите секцию "Конфигурация" в начале файла
3. Измените следующие параметры:

```swift
/// Владелец/название репозитория (например "username/repo")
private let repository = "OWNER/REPO" // ← ЗАМЕНИТЕ НА ВАШ РЕПОЗИТОРИЙ

/// Название папки в репозитории
private let folderName = "KnightsGame" // ← НАЗВАНИЕ ПАПКИ НА GITHUB
```

### Пример:
Если ваш репозиторий: `https://github.com/ivan123/knights-game`
И папка в репозитории: `KnightsGame`

Тогда:
```swift
private let repository = "ivan123/knights-game"
private let folderName = "KnightsGame"
```

## 📋 Как это работает

### При запуске приложения:

1. ✅ **Приложение ВСЕГДА открывает локальную версию** (Bundle или Documents)
2. 🔄 **В фоне** начинается проверка обновлений из GitHub
3. 📡 Если нет интернета — обновление пропускается молча
4. ⬇️ Если есть интернет — скачиваются файлы во временную папку
5. 📁 После успешной загрузки — временная папка переименовывается в `KnightsGame` в Documents
6. 🎮 **Новая версия применяется при следующем запуске**

### Приоритет загрузки файлов:

1. **Documents/KnightsGame** (если есть) — обновлённая версия
2. **Bundle/KnightsGame** (fallback) — версия из установки

## 🔍 Логи

В консоли Xcode вы увидите:

```
🔄 Начинаем фоновое обновление...
📋 Получен список файлов: 15 файлов
✅ Скачан: index.html
✅ Скачан: styles/main.css
✅ Скачан: js/game.js
...
⬇️ Файлы скачаны во временную папку
✅ Обновление завершено успешно
📂 Загружаем из Documents (обновлённая версия)
```

Или если нет интернета:
```
📡 Нет подключения к интернету, обновление пропущено
📦 Загружаем из Bundle (версия по умолчанию)
```

## 🎯 Требования выполнены

- ✅ Приложение ВСЕГДА открывает файлы мгновенно (из Bundle или Documents)
- ✅ Фоновое обновление из GitHub (ветка main)
- ✅ Скачивание во временную папку с переименованием
- ✅ Без индикатора загрузки
- ✅ Новая версия применяется при следующем запуске
- ✅ Нет интернета = нет попыток обновления
- ✅ Работает с публичными репозиториями

## 📝 Структура репозитория

Ваш GitHub репозиторий должен иметь структуру:

```
OWNER/REPO (ветка main)
└── KnightsGame/
    ├── index.html
    ├── styles/
    │   └── main.css
    ├── js/
    │   └── game.js
    └── assets/
        └── images/
```

Все файлы из папки `KnightsGame` будут скачаны рекурсивно.
