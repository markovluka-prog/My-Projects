Глава 1

Creative an Invention

локальная среда Лука AI работа с проектами инструменты

О чём этот проект

Creative an Invention — это попытка собрать не просто чат и не просто редактор, а рабочую среду, которая умеет держать рядом мысли, проекты, файлы, поиск и действия. По материалам проекта видно, что он задуман как личный центр управления, где пользователь разговаривает не с безличной системой, а с локальным помощником по имени Лука AI.

Внутри проекта особенно важен не внешний блеск, а характер взаимодействия: помощник должен быть кратким, прямым, русскоязычным и не перекладывать работу на пользователя, если её можно сделать самому инструментами.

Прямой фрагмент

«Ты Лука AI. Это агентный системный промпт.

Ты локальный агент по проектам, коду, файлам, исследованиям и веб-поиску.

Если задача многошаговая, действуй как агент: анализируй, используй инструмент, оценивай результат, завершай.»

Как он думает о помощнике

В этом проекте помощник не должен быть общим разговорным ботом. Ему заранее задают роль: работать по-русски, говорить кратко и прямо, не болтать без пользы, не повторять вопрос, не выдавать шаблонных вступлений и не прятаться от инструментов там, где нужен реальный шаг.

Даже в быстрых ответах система жёстко удерживает характер: если вопрос простой, отвечай просто; если вопрос про проект, опирайся на контекст проекта; если ответ должен быть точным, не уходи в рассуждения.

В проект уже заложены роли

  • быстрый режим для коротких смысловых ответов;
  • агентный режим для многошаговых задач с инструментами;
  • роутер, который сначала выбирает способ работы: `fast`, `direct` или `agent`;
  • память о пользователе, имени, стиле общения и рабочей директории;
  • отдельные навыки: thinking, planning, editing, project summary, patent, coding, memory.

Формат действия

Внутри проекта заложена очень ясная идея: помощник должен говорить либо финальным ответом, либо одним точным вызовом инструмента. Этот фрагмент особенно хорошо показывает характер всей среды:

{
"type":"final","message":"..."
}

{
"type":"tool",
"tool":"read_file",
"args":{"path":"..."},
"reason":"..."
}

Проект как среда для высказывания

Есть важная внутренняя фраза: «Можешь просто описать проект своими словами, а я сам соберу из этого описание, идею и набросок концепта». В ней чувствуется вся философия Creative an Invention. Пользователь не обязан приходить с идеальным документом. Он может принести сырой текст, а среда сама поможет превратить его в структуру.

Отсюда вырастает и более широкая идея: это не просто ассистент по готовому коду, а инструмент, который помогает из хаоса сделать проектную форму.

Глава 2

LIT GPT

маленький чат-бот словарь слов схожесть по буквам Pythonista

Честный маленький GPT

В материалах LIT GPT проект описан очень прямо: «Простой GPT-подобный чат-бот, написанный с нуля на Python без ML-библиотек». Это важная отправная точка. Перед нами не попытка имитировать огромную модель, а желание самому понять, из чего вообще может рождаться ответ.

Поэтому и архитектура у проекта прозрачная: словарь, обучение на новых словах, вес слова, алгоритм схожести и интерфейс, где пользователь просто пишет сообщение и получает короткий ответ.

Прямой фрагмент

«1. learn(text) — добавляет новые слова из текста в словарь.

2. answer(text) — для каждого слова вопроса считает вес и схожесть со всеми словами словаря, возвращает самое близкое слово.

3. Вес слова растёт каждый раз, когда оно используется в запросе.»

Что здесь особенно интересно

Проект строится не вокруг модного слова «AI», а вокруг ясного механизма. Есть словарь слов, который можно пополнять. Есть ранжирование, которое поднимает использованные слова вверх. Есть модель сходства, которая сравнивает слова по позициям букв. Есть интерфейс, где всё это выглядит как один простой диалог.

Такой подход делает LIT GPT особенно ценным: здесь виден маршрут мышления. Автор не закрывает устройство чёрным ящиком, а вытаскивает его наружу и делает обозримым.

Фрагмент кода: рост веса

def promote_one(self, word):
    if word not in self.weight:
        self.weight.append(word)
        self.save()
        return

    idx = self.weight.index(word)
    if idx == 0:
        return

    self.weight[idx], self.weight[idx - 1] = self.weight[idx - 1], self.weight[idx]
    self.save()

Что делает этот кусок важным

Здесь буквально видно, как слово поднимается вверх при повторном использовании. Это почти тактильная модель памяти: не сложная статистическая машина, а ручное продвижение слов по шкале значимости. Для большой промышленной модели это слишком просто. Но для авторского проекта это именно то, что нужно: механизм можно видеть и понимать.

Запас роста внутри проекта

По документам видно, что автор уже сам наметил следующие шаги: сделать ответы из нескольких слов, добавить контекст между вопросами, обновлять веса не только при ответе, но и при обучении, а затем перенести проект из Pythonista в более универсальную среду. Рядом лежат крупные языковые корпуса на десятки тысяч слов, значит проект уже смотрит дальше своего первого рабочего состояния.

Глава 3

Vocabulary Race

словесная бродилка 2–4 игрока боты 48 клеток

Игра, которая звучит как партия за столом

В собственных материалах Vocabulary Race описана просто и очень ясно: это настольная словесная игра с элементами бродилки для 2–4 игроков на одном устройстве. Игроки составляют слова из букв карточки, двигают фишки по замкнутому маршруту и пытаются первыми дойти до победной клетки.

Но дальше описание быстро показывает, что проект давно перестал быть одной идеей. У него есть загрузочный экран, диалог выбора игроков, экран-заглушка от подглядывания, таймер, спецкарты, боты и даже длинный список анимаций и исправлений.

Поле на 48 клеток собрано как замкнутый маршрут. Внутри проекта отдельно зафиксировано, что координаты маршрута выставлялись вручную.

Прямой фрагмент

«1. Тянется карточка из колоды.

2. Если спецкарточка — сохраняется у игрока, тянется следующая.

3. Если у игрока есть спецкарточки — выбор: использовать или играть обычно.

4. Обычная игра → экран задания, 60 секунд, составить N слов.

5. Успех → +N очков, фишка вперёд; провал → -N очков, фишка назад.»

Структура экранов

Экран Зачем он нужен

Что уже встроено в игру

  • боты умеют играть автоматически и используют спецкарты;
  • сложность бота задаётся явными вероятностями успеха;
  • машинки поворачиваются по направлению следующей клетки;
  • после провала игроку показывают слова, которые можно было составить;
  • для людей есть авто-продолжение результата с обратным отсчётом.

Почему это уже больше, чем упражнение

Vocabulary Race держится не на одном только словаре. Внутри проекта уже есть ритм партии. Есть момент ожидания перед ходом. Есть напряжение таймера. Есть экран результата и экран победы. Есть две спецкарты, которые вносят в игру случайность и переворот позиции: «Гонщик» и «Судья».

Очень живой деталью выглядит экран-заглушка — простое, но умное решение для игры на одном устройстве, когда нужно скрыть карточку от других игроков.

Прямой фрагмент

«Экран-заглушка — “Синий игрок, нажмите чтобы начать” (защита от подглядывания).»

«Подсказка после провала показывает слова, которые можно было составить.»

Глава 4

Рыцари и Замки

пошаговая стратегия кампания 7 уровней ИИ тёмный средневековый мир
Обложка проекта сразу задаёт тон: мрачный фон, золото, камень, бой и общее чувство тяжёлой тактики.

Стратегия с собственной лестницей сложности

Рыцари и Замки — это пошаговая стратегия в средневековом сеттинге, где игрок двигает отряды по сетке, бьётся с ИИ и проходит сюжетную кампанию. Но важнее другое: проект уже развил целую шкалу характера противника — от почти случайного «Новичка» до адаптивной «Легенды».

Материалы проекта соединяют несколько слоёв сразу: описание сетки и интерфейса, музыкальную среду, список юнитов, кампанию из трёх глав и огромный блок AI-пресетов, где каждый уровень сложности буквально прописан как личность.

Прямой фрагмент

«Новичок — ходит почти случайно, не понимает угроз, игнорирует позицию.»

«Воин — понимает фланги, контролирует карту, умеет обороняться.»

«Легенда — финальный босс. Подстраивается под игрока. Используется только 1 раз.»

Почему это важно

Обычно в небольшом авторском проекте сложность означает просто рост чисел. Здесь всё иначе. Даже на уровне документов видно, что ИИ мыслят как разные типы противников: один слеп к угрозам, другой любит контроль территории, третий использует фланги, четвёртый способен жертвовать юнитами ради позиции. Это уже не голый параметр, а драматургия боя.

Кампания и названия заданий

Три главы кампании уже названы так, будто это полноценная тёмная сага: «Первые шаги», «Морской ветер», «Разлом». Внутри каждой есть конкретные задания: «Патруль у ворот», «Двойная засада», «Капитан Соколин», «Абордаж в тумане», «Последняя битва», «Цена доверия», «Второй шанс».

Из этих названий видно, что игра стремится быть не просто сеткой с юнитами, а миром с собственным ритмом, предательством, восстановлением, морем и личными именами.

Глава 5

Звезда Луки

изобретение трансформер без правил рынок патент
Рендер объекта показывает главное: это не заранее заданный герой, а цельная система сочленений, которая меняет характер от поворота к повороту.
Реальный прототип подтверждает, что идея существует не только как текст. Объект уже собран физически.

Трансформер без правил

Концепт проекта формулирует проблему очень точно: большинство трансформеров имеют фиксированный сценарий трансформаций, ребёнок повторяет, а не придумывает. Звезда Луки отвечает на эту проблему радикально: это «трансформер без правил», цельный объект на шаровых шарнирах, где нет инструкции и есть бесконечные варианты.

Главное здесь не одна красивая форма, а принцип. Базовая позиция — плоский крест. Дальше один и тот же объект может стать роботом, птицей, машиной, крабом, кольцом или танцором. Это не игрушка, которая показывает одну тайну; это игрушка, которая открывает класс действий.

Прямой фрагмент

«Звезда Луки — трансформер без правил. Цельный объект на шаровых шарнирах: одно движение — скрутил, повернул — и форма изменилась. Нет инструкции — есть бесконечные варианты.»

Особенно важно, что все формы достигаются без снятия деталей. В проекте это постоянно повторяется как главная ось идентичности: объект цельный, а меняется только его положение.

Устройство конструкции

Дизайн описан очень подробно. Каркас образует два условных квадрата: внутренний пустой проём и внешний периметр. Четыре чёрных коннектора соединяют их, огибая центр и не касаясь друг друга. По периметру сидят четыре бирюзовых блока, на которых держатся шарниры и рычаги. Форма определяется двумя независимыми параметрами: положением рычагов и опциональным скручиванием секций.

Даже без фотографии по тексту видно, что автор видит конструкцию пространственно и геометрически, а не как грубую игрушечную сборку.

Глава 6

Механика Ниндзя

личная теория пустая голова рефлексы поток

Открывающий фрагмент

«Механика Ниндзя — это состояние пустой головы, в котором подсознание берёт управление и делает всё лучше чем сознание.»

«Удача приходит не случайно — она приходит когда перестаёшь мешать себе думать.»

Что это за теория

Механика Ниндзя — личная теория о том, что удача и высокая точность приходят тогда, когда сознательный шум отступает, а управление берут на себя рефлексы и подсознание. По документам видно, что проект не застрял на уровне сырой идеи: у него есть описание, статья для широкой аудитории, список примеров, методы входа, три режима применения и зафиксированные эксперименты для будущей проверки.

Здесь особенно интересно, что автор называет это не философией и не просто настроением, а механикой. То есть пытается поймать переход между состояниями и дать ему рабочий язык.

Три слоя

  • Шум — анализ, сомнения, страх ошибки, заблокированные рефлексы.
  • Тишина — мысли выгнаны, сознание отходит в сторону, подсознание получает прямой доступ к действию.
  • Поток — результат: скорость, точность и ощущение удачи.

Два метода входа

  • Пение в Ниндзя — петь без мыслей, занять речевой центр и заглушить внутренний диалог.
  • Концентрация на ничто — выгонять мысли напрямую, входить за 1–2 минуты.

Прямой фрагмент

«Подсознание — это рефлексы: накопленные паттерны которые работают быстро, точно и без участия сознания. Они уже знают что делать. Проблема в том что сознание постоянно перекрывает им дорогу — анализирует, сомневается, мешает.»

Три режима использования

В теории отдельно зафиксированы три способа применения. «Момент» — войти в состояние и сразу получить лучший результат. «Обучение» — войти, сделать действие, выйти, чтобы тело записало паттерн глубже. «Генерация» — отдать инициативу подсознанию, чтобы оно само придумало новое действие.

Этот третий режим особенно выделяет проект. Подсознание здесь не только быстрее исполняет, но и может выступать автором движения.

Прямой фрагмент из статьи

«Воздушный змей. Сильный ветер, сложные условия. Я вошёл в состояние — и змей буквально затянуло прямо в руки.»

«Кунг-фу на пляже. В состоянии Ниндзя тело само давало мне упражнения — странные, которые я бы не придумал.»

Эксперименты

Эксперимент Что проверяет

Честная часть теории

В документах особенно ценен не только оптимизм, но и открытые вопросы. Проект признаёт, что эффект выхода ещё не объяснён, что научный консенсус отсутствует и что нужны эксперименты, публикация статьи и прояснение механизма отката.

Эта честность делает Механику Ниндзя не мистическим лозунгом, а живой исследовательской гипотезой, в которой уже есть и внутренний язык, и список того, что нужно проверить.

Предупреждение из статьи

«После выхода из состояния примерно в 75% случаев наступает временная полоса невезения. Почему — я пока не знаю.»

Связи с другими идеями

Проект сам сопоставляет себя с Unconscious Thought Theory, спортивной «зоной», flow, wu wei и beginner's mind. Но важная мысль документа в другом: автор считает, что нигде не было одновременно и конкретного метода входа, и режима обучения через подсознание. Именно на этом Механика Ниндзя пытается настоять как на собственной новизне.

Проза как движение

Главная сила текста в том, что он всё время толкает вперёд. Ричард прыгает на мачту, корабль разворачивается, якоря цепляются, абордажная доска падает, щит вышибает меч, гарпун закручивается вокруг мачты. Это мир, который не стоит на месте. Он живёт через манёвр и столкновение.

Даже сцены тренировки выглядят как испытание характера. Зантио ставят капитаном не ради титула, а как проверку: сможет ли он стать ниндзей.

«— Стань капитаном, — сказал Ричард.

«РИЧАРД: Стань капитаном.»

«ЗАНТИО (удивлённо): Я?»

«РИЧАРД: Ты. Все будут тебя слушаться. Докажи, что можешь быть ниндзей.»

Почему наличие сценария так важно

Когда у мира уже есть и проза, и сценарий, он перестаёт быть просто «текстом в работе». Он начинает существовать как многослойная сцена. В прозе можно чувствовать широту и внутренний голос, а в сценарии — кадр, монтаж, паузу, реплику и ритм.

Это делает Ниндзи одним из самых кинематографичных проектов во всей книге.

Глава 8

Пружинная гравитация Луки

альтернативная физика формулы антигравитон слабые места

Открывающий фрагмент

«Гравитация работает не как у Ньютона, а наоборот — как пружина: чем дальше объект от центра ядра, тем сильнее его притягивает. При выходе за границу поля — притяжение мгновенно падает до нуля.»

О чём эта гипотеза

Пружинная гравитация Луки — это не просто набор смелых слов, а целая альтернативная модель, у которой уже есть уровни теории, формулы, свойства поля, антигравитон, космологический слой, эксперименты и, что особенно важно, честный документ со слабыми местами.

Здесь особенно заметен исследовательский темперамент автора: идея не просто провозглашается, а обрастает определениями, параметрами, исправлениями, удалёнными формулами и списком того, что пока не сходится.

Формула 1 — сила притяжения меньшего объекта если k ≤ k_макс: F = k / k_макс * m1 если k > k_макс: F = 0
Формула 2 — максимальный радиус поля k_макс = √(m² + r²)

Основные понятия

  • ядро — группа протонов, объединённых в источник поля;
  • поле гравитации — зона вокруг ядра с конечной границей;
  • гравитон — частица, усиливающая поле;
  • антигравитон — электрон или газ с ним, нейтрализующий поле;
  • тёмная материя — скопление гравитонов;
  • тёмная энергия — антигравитон на уровне Вселенной.

Свойства поля

По этой теории поле повторяет форму объекта, движется вместе с ним, но не вращается вместе с ним, может перекрываться с другими полями, проходит сквозь вещество, мгновенно обрывается на границе и в принципе поддаётся нейтрализации антигравитоном.

Из этого рождаются не только формулы, но и почти инженерные образы: палка с двумя кранами для газов, гравитационная бомба, силовое поле вокруг объекта.

Прямой фрагмент

«У каждого объекта есть конечная зона притяжения — за ней он не влияет на окружающее.»

«Вселенная — не бесконечно равномерная, а живая структура с переменным внутренним давлением гравитации.»

Эксперименты и проверки

Почему этот раздел важен

В проекте есть редкая честность: неудачные совпадения не прячутся. Автор прямо фиксирует, что одна проверка не сработала, другая дала частичное совпадение, третья поставила новый вопрос. Это делает главу сильнее, а не слабее. Перед нами не мнимая завершённость, а настоящая работа мысли.

Сила Пружинной гравитации не в том, что гипотеза уже подтверждена, а в том, что автор довёл мысль до формул, экспериментов, пересмотра формул, космологии, инженерных следствий и списка критических возражений. Это редкая дисциплина для идеи такого масштаба.
