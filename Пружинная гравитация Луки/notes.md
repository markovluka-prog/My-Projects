## 2026-03-12

Основная идея из презентации:
- Гравитация как пружина — нарастает к краю поля, обнуляется за полем
- Формула: `a = (bm * g) / 2 * k^n`, где g = 2.0006
- Ядро = группа протонов, любой объект = своё ядро
- Антигравитон = электрон (e-), нужно отрицательных ионов в 2 раза больше положительных
- Устройство: палка с двумя кранами — гравитон (вверх) + антигравитон (вниз)
- Вселенная = одно огромное ядро из гуглов протонов, разное давление внутри создаёт объекты

Из Python-кода:
- Код написан для Pythonista / Math Worker на iOS
- Реализует проверку по формуле: если объект за пределами поля — возвращает False
- Формула в коде: `a = (b * g) / 2 * k` (без степени n)
- Поле в коде: `k_maks = b ** g / g`
- Есть меню: "Сила гравитации" и "Поле гравитации"

Нужно разобраться: формула в коде и в презентации отличаются. Спросить Луку.

## 2026-03-13

Сессия с Claude — разбор формул и слабых мест.

Что выяснили:
- g убрана из формулы 1 — давление = b/2 * k
- Единица Л (Лука) = кг·м — новая единица для формулы 1
- Формула 2 исправлена: k_макс = b^(1/g) — старая давала поля больше Вселенной
- Новые значения k_макс: Земля = 2.4×10¹² м, Солнце = 1.4×10¹⁵ м, Протон = 4×10⁻¹⁴ м

Главный нерешённый вопрос — орбиты. При нарастающей гравитации планеты не могут стабильно орбитировать: формула предсказывает скорости в 10⁹ раз больше реальных. Это самое слабое место теории.
